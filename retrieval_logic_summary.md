# Retrieval Logic Summary

- Article ingestion reads `content/sources.json`, fetches each page, strips scripts/styles/nav/header/footer/form/media with Cheerio, extracts `main` text or body text, chunks content at about 2,600 characters with 300-character overlap, embeds each chunk, and stores it in `brunel_documents`.
- Dataset ingestion reads `content/datahub-datasets.json`, downloads the public Datahub-data ZIP, finds each mapped folder, parses `.xlsx` workbooks, chooses the first sheet whose name includes `analysis` as the analysis sheet, stores analysis rows in `brunel_dataset_rows`, parses metadata sheets into file/fact metadata, and parses raw sheets into `brunel_dataset_facts` where a wide year-header structure is detected.
- Dataset summaries are embedded in `brunel_dataset_summaries` and include post URL, workbook name, analysis sheet, columns, row counts, raw fact counts, metadata and sample rows/facts.
- Chat request order is: small-talk pre-check, catalogue/show-more handler, NEET explanation shortcut, deterministic statistical backend for calculation/count/method style requests, then OpenAI embedding retrieval across article chunks, dataset summaries, analysis rows, and optionally raw facts.
- Ordinary numerical lookup prompt rules say to use article context first, then analysis dataset rows if the value is not in the article. Raw facts should be exposed only for calculations, counts, methods or detailed raw-data questions.
- Multi-topic retrieval decomposes recognised concepts such as GDP, GVA, productivity, employment, wages, skills, NEET, emissions, energy, housing and sectors. It builds concept-specific and synonym queries, merges results, and asks the model to count topic coverage correctly using a confirmed topic-source map.
- Catalogue responses are deterministic and do not call retrieval/LLM. They read the source registry directly, list confirmed source records only, render Markdown links, suppress duplicate bottom source cards, and maintain show-more context only from the immediately previous assistant catalogue response.
